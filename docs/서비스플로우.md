# QA-Server 서비스 플로우 다이어그램

---

## 1. 전체 서비스 플로우

```mermaid
flowchart TD
    A([사용자]) --> DASH[전체 대시보드\n메인 페이지]
    DASH -->|카드 클릭| B[프로젝트 진입]
    DASH -->|사이드바 선택| B
    B --> C{탭 선택}

    C --> SU[Test Suite]
    C --> D[케이스 관리]
    C --> E[자동 실행]
    C --> F[실행 히스토리]
    C --> AN[실행 분석]
    C --> CH[변경 이력]
    C --> G[테스트결과서]
    C --> H[배포결과서]

    SU --> SU1[스위트 생성 / 편집]
    SU --> SU2[케이스 + 플로우 체크박스 선택]
    SU --> SU3[기본 스위트 설정]
    SU2 --> SU4[스위트 적용\nactiveSuiteId 저장]
    SU4 --> D
    SU4 --> E

    D --> D1[카테고리 생성]
    D --> D2[케이스 입력 / 엑셀 임포트]
    D --> D3[히스토리에서 불러오기]
    D2 --> D4[저장 → 변경 이력 자동 기록]

    E --> E1[케이스 + 플로우 선택]
    E --> E2[Base URL 입력]
    E1 & E2 --> E3[실행 시작]
    E3 --> E4[서버 HTTP 요청 실행]
    E4 --> E5{결과}
    E5 -->|Pass| E6[Pass 기록]
    E5 -->|Fail| E7[Fail 기록]
    E6 & E7 --> E8[실행 완료 → test_runs 저장 불변]
    E8 --> E9[Discord / Slack 알림 전송]

    F --> F1[실행 목록 검색 / 필터]
    F1 --> F2[실행 선택]
    F2 --> F3[케이스별 결과 확인\nPass/Fail · 카테고리 필터]
    F2 --> F6[레이블 편집]
    F2 --> F7[댓글 추가]
    F2 --> F8[설정 불러오기 → 케이스관리/자동실행 복원]

    AN --> AN1[기간 필터 설정]
    AN1 --> AN2[Pass율 추이 확인]
    AN1 --> AN3[케이스별 실패 순위 확인]

    CH --> CH1[생성/수정/삭제 필터]
    CH1 --> CH2[변경 전·후 상세 확인]

    G --> G1[커버 페이지 작성]
    G --> G2[케이스별 actual/notes 입력]

    H --> H1[배포 커버 작성]
    H --> H2[배포 점검 케이스 기록]
    H --> H3[AI 분석 → LLM 리포트]
```

## 2. 대시보드 플로우

```mermaid
flowchart TD
    진입([앱 최초 진입]) --> DASH[대시보드\n전 프로젝트 카드 표시]
    DASH --> PERIOD[기간 필터 선택\n전체·7일·30일·90일·직접설정]
    PERIOD --> RELOAD[카드 + 차트 재조회]

    DASH --> MORE{프로젝트 > 6개?}
    MORE -->|Yes| BTN[더보기 버튼\n6개씩 추가 표시]
    MORE -->|No| CARDS[전체 카드 표시]

    DASH --> CLICK[카드 클릭]
    CLICK --> SELECT[프로젝트 선택\n히스토리 탭으로 이동]

    DASH --> COMPARE{프로젝트 2개 이상\n+ 실행 이력 있음?}
    COMPARE -->|Yes| CHARTS[비교 차트 자동 표시\nPass율 바차트\n실행횟수 바차트\n케이스 레이더차트]
```

## 3. 자동 실행 시퀀스

```mermaid
sequenceDiagram
    participant U as 사용자
    participant FE as Frontend (React)
    participant API as FastAPI
    participant DB as PostgreSQL
    participant WH as Webhook (Discord/Slack)

    U->>FE: 케이스 선택 + Base URL 입력
    U->>FE: 실행 버튼 클릭
    FE->>API: POST /api/runs
    API->>DB: INSERT test_runs (status=running)
    API-->>FE: run_id 반환

    loop 폴링 (1초 간격)
        FE->>API: GET /api/runs/{id}
        API-->>FE: status, done, total, fail
    end

    Note over API,DB: 백그라운드 실행
    loop 각 케이스
        API->>API: HTTP 요청 실행
        API->>DB: done/fail 카운트 업데이트
    end

    API->>DB: case_results, flow_results, mgr_snapshot 저장 (불변)
    API->>DB: status = done
    API->>WH: 알림 전송 (설정된 경우)

    FE->>API: GET /api/runs/{id}
    API-->>FE: 최종 결과
    FE->>U: 결과 표시
```

## 4. 케이스 변경 이력 자동 기록

```mermaid
flowchart LR
    SAVE[케이스 관리 저장\nPOST /api/qa/data] --> READ[기존 스냅샷에서\nold_cases 추출]
    READ --> OVERWRITE[qa_snapshots 업데이트]
    READ --> DIFF[new_cases와 diff 비교]

    DIFF --> C{변경 유형}
    C -->|새 ID 추가| CREATED[case_histories\naction=created\nafter=케이스 전체]
    C -->|추적 필드 변경| UPDATED[case_histories\naction=updated\nbefore+after 전체]
    C -->|기존 ID 삭제| DELETED[case_histories\naction=deleted\nbefore=케이스 전체]

    CREATED & UPDATED & DELETED --> LOG[case_history.log 기록]
```

## 5. 히스토리 불러오기 (Cross-page 동기화)

```mermaid
flowchart LR
    subgraph HistoryPage
        H1[히스토리 선택]
        H2[불러오기 클릭]
        H1 --> H2
    end

    subgraph CaseManagerPage
        C1[히스토리에서 불러오기]
        C2[실행 목록 모달]
        C3[케이스 필터 적용\n+ pendingRunRestore 설정]
        C1 --> C2 --> C3
    end

    subgraph ZustandStore["Zustand Store"]
        Z1["pendingRunRestore\ncaseIds / flowIds / baseUrl"]
    end

    subgraph AutoRunPage
        A1[useEffect 감지]
        A2[selected 복원]
        A3[cases 탭 이동]
        A1 --> A2 & A3
    end

    H2 --> A2 & A3
    C3 --> Z1 --> A1
```

## 6. 알림 전송 플로우

```mermaid
flowchart LR
    RUN[실행 완료 / 실패] --> FETCH[notification_configs 조회\nenabled=true + 해당 event 구독]
    FETCH --> LOOP[각 config 순회]
    LOOP --> TYPE{type}
    TYPE -->|discord| DISCORD["Discord Webhook\nEmbed 카드 전송\n색상: 초록/주황/빨강"]
    TYPE -->|slack| SLACK["Slack Webhook\ntext 메시지 전송"]
    DISCORD & SLACK --> LOGFILE[runs.log 기록]
```

## 7. 테스트 플로우 실행 (stop-on-fail)

```mermaid
flowchart TD
    S([플로우 실행 시작]) --> Step1[스텝 1 실행]
    Step1 --> R1{결과}
    R1 -->|Pass| Step2[스텝 2 실행]
    R1 -->|Fail| Skip2[스텝 2 스킵]
    Step2 --> R2{결과}
    R2 -->|Pass| Step3[스텝 3 실행]
    R2 -->|Fail| Skip3[스텝 3 스킵]
    Step3 --> R3{결과}
    R3 --> Done([플로우 결과 저장])
    Skip2 --> Skip3 --> Done
```

## 8. Test Suite 동기화 플로우

```mermaid
flowchart LR
    subgraph SuitePage
        S1[스위트 선택]
        S2[케이스·플로우 토글]
        S3[적용 버튼 클릭]
        S1 --> S2 --> S3
    end

    subgraph ZustandStore["Zustand Store"]
        Z1["activeSuiteId\n(number | null)"]
    end

    subgraph CaseManagerPage
        C1[activeSuiteId useEffect]
        C2[스위트 case_ids 로드]
        C3[케이스 필터링\n배너 표시]
        C1 --> C2 --> C3
    end

    subgraph AutoRunPage
        A1[activeSuiteId useEffect]
        A2[case_ids → selected 세팅]
        A3[flow_ids → selectedFlowIds 세팅]
        A4[배너 표시]
        A1 --> A2 & A3 --> A4
    end

    S3 --> Z1
    Z1 --> C1
    Z1 --> A1
```
