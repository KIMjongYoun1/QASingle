# Single_QA_Tools — 서비스 소개서

---

## 개요

**Single_QA_Tools**는 소규모 개발팀 또는 개인 QA 담당자가 테스트 케이스 관리부터 자동화 실행, 배포 점검까지 하나의 인터페이스에서 수행할 수 있는 통합 QA 관리 도구입니다.

---

## 핵심 기능

### 1. 케이스 관리 (Case Manager)

- 테스트 케이스를 카테고리 단위로 구조화 관리
- 각 케이스에 HTTP 메서드 / 엔드포인트 / 헤더 / 파라미터 / 바디 등 실행 파라미터 직접 입력
- Pass / Fail / 보류 상태 수동 기록
- 연동규격서 엑셀 포맷 임포트 지원
- 히스토리에서 이전 실행 설정 불러오기 — 케이스 필터를 원클릭으로 복원

### 2. 자동 실행 (Auto Run)

- 선택된 개별 케이스 + 테스트 플로우를 순차·병렬 혼합 실행
- Base URL 지정 후 실제 HTTP 요청을 서버가 대신 수행 (사람이 실행 버튼을 누르는 트리거 방식)
- 실행 결과(actual / pf / notes)를 케이스 단위로 기록, 이후 불변 저장
- 히스토리 불러오기로 이전 실행 케이스 선택 즉시 복원

### 3. 테스트 플로우 (Test Flow)

- 업무 흐름 단위로 케이스를 순서대로 묶은 플로우 정의
- 스텝 실패 시 이후 스텝 자동 건너뜀 (stop-on-fail)
- 플로우 단위 실행 결과 별도 저장

### 4. 실행 히스토리 (History)

- 모든 자동 실행 이력을 시계열로 보관
- 실행 단위 레이블 편집 가능 (나머지 결과는 불변)
- 실행별 상세: 개별 케이스 결과 + 플로우 스텝 결과
  - Pass / Fail 필터, 카테고리 필터 지원
  - 각 케이스 행 클릭 시 actual / notes 인라인 확인
- 실행 단위 댓글 추가 (append-only)
- 왼쪽 패널: 실행 목록 검색 + 전통과 / 실패포함 필터
- 히스토리 설정 → 케이스관리 / 자동실행 양방향 복원

### 5. 테스트결과서 (TST)

- 최종 테스트 결과를 문서 형식으로 정리
- 커버 페이지(프로젝트명 / 환경 / 작성자 / 승인자) + 케이스별 결과 기록
- 사이드바 케이스 트리로 특정 케이스 바로 이동

### 6. 배포결과서 (DEP)

- 배포 점검 결과 기록 및 이력 관리
- 배포 버전 / 환경 / 배포 유형(정기 / 긴급 / 롤백 등) 분류
- LLM AI 분석: 배포 이력을 기반으로 패턴 및 위험 포인트 분석

## 기술 스택

| 영역 | 기술 |
|---|---|
| Frontend | React 18, TypeScript, Vite, Tailwind CSS v4, shadcn/ui, Zustand, Sonner |
| Backend | FastAPI (Python), SQLAlchemy, PostgreSQL |
| 실행 방식 | 사용자 트리거 (Human-in-the-loop) |
| AI 분석 | Anthropic Claude API |

---

## 아키텍처 요약

```
Browser (React SPA)
  └── Zustand 스토어 (전역 상태: 선택된 케이스, 프로젝트, pendingRunRestore)
        │
        ▼
FastAPI REST API
  ├── /projects       — 프로젝트 CRUD
  ├── /qa             — QA 스냅샷 load/save (mgr/tst/dep 통합)
  ├── /runs           — 자동 실행 생성·조회·폴링
  ├── /flows          — 테스트 플로우 CRUD
  ├── /deploy         — 배포결과서 이력
  └── /analysis       — LLM 분석 (Claude API)
        │
        ▼
PostgreSQL
  (projects, qa_snapshots, deploy_histories, test_flows, test_runs, run_comments)
```

---

## 사용 시나리오

1. **신규 프로젝트 셋업**: 프로젝트 생성 → 카테고리 정의 → 케이스 입력 (또는 엑셀 임포트)
2. **자동 실행**: 케이스 선택 + Base URL 지정 → 실행 → 결과 자동 기록
3. **이력 재활용**: 히스토리에서 이전 실행 불러오기 → 케이스 관리/자동실행 탭에 즉시 적용
4. **문서 작성**: 실행 완료 후 테스트결과서에 actual / notes 보완 → 배포결과서 작성
5. **분석**: AI 분석으로 배포 패턴 리포트 확인

---

## 불변성 원칙

실행이 완료된 결과(case_results, flow_results, mgr_snapshot)는 절대 수정되지 않습니다.  
`label`만 편집 가능하며, 댓글은 추가만 허용됩니다.  
이로써 실행 이력의 감사 가능성(Audit trail)을 보장합니다.
